# 遥操作与数据采集

> 本文档覆盖：SpaceMouse 遥操作原理、数据采集完整流程、Zarr 数据格式、常见问题排查。

## 整体架构

```
SpaceMouse 6D 输入 + 按钮
       ↓
SpacemouseAgent (缩放 / 轴重映射 / 死区过滤 / 夹爪切换)
       ↓ action [dx, dy, dz, dr, dp, dy] + gripper_action
XArmEnv (delta_eef, speed=400)
       ↓ 执行动作
并行采集: 末端位姿 + 关节角 + 力传感器 + 双相机 RGBD
       ↓
Collector → Zarr 数据集 (Blosc 压缩) + 可选 MP4 视频
```

## 一、SpaceMouse 遥操作

### 1.1 坐标映射

SpaceMouse 的 6D 原始输入经过 `SpacemouseAgent` 缩放映射后发送给机械臂：

| SpaceMouse 轴 | 映射到 | 默认缩放 |
|---------------|--------|---------|
| 平移 X | 机械臂 X | `× translation_scale` (默认 5.0) |
| 平移 Y | 机械臂 Y | `× translation_scale` (默认 5.0) |
| 平移 Z | 机械臂 Z | `× z_scale` (默认同 translation_scale) |
| 旋转 roll | 机械臂 **-pitch** | `× rotation_scale` (默认 0.004) |
| 旋转 pitch | 机械臂 **roll** | `× rotation_scale` (默认 0.004) |
| 旋转 yaw | 机械臂 yaw | `× rotation_scale` (默认 0.004) |

> **为什么 roll/pitch 做了交换？** 因为 XArm6 Home 位姿的 yaw = -π/2（夹爪旋转了 90°），所以 SpaceMouse 的 roll/pitch 需要交叉映射到机械臂的 pitch/roll。如果修改了 Home 位姿的 yaw 角，这个映射需要对应调整。

### 1.2 灵敏度参数

| 参数 | 默认值 | 作用 | 调整建议 |
|------|--------|------|---------|
| `translation_scale` | 5.0 | 平移灵敏度 (mm/tick) | 精细任务降到 2~3 |
| `rotation_scale` | 0.004 | 旋转灵敏度 (rad/tick) | 精细任务降到 0.002 |
| `deadzone` | 0.0 | 死区阈值，低于此值视为 0 | 手抖严重时设 0.05~0.15 |

### 1.3 按钮功能

**teleop_demo.py（纯遥操作）：**

| 操作 | 功能 |
|------|------|
| SpaceMouse 左键 | **切换**夹爪开/关（按一次切换一次） |
| 键盘 `q` / `Ctrl+C` | 退出 |
| 键盘 `r` | 复位到 Home |
| 键盘 `o` | 手动打开夹爪 |
| 键盘 `c` | 手动关闭夹爪 |

**collect_data.py（数据采集）：**

| 阶段 | SpaceMouse 左键 | SpaceMouse 右键 |
|------|----------------|----------------|
| 等待开始（空格前） | 张开夹爪 (+40) | 关闭夹爪 (-40) |
| 录制中 | 张开夹爪 (→ 840) | 关闭夹爪 (→ 0) |

### 1.4 快速启动

```bash
# 基础遥操作（不保存数据）
python scripts/teleop_demo.py

# 调整灵敏度
python scripts/teleop_demo.py --trans-scale 3 --rot-scale 0.003 --deadzone 0.1

# 启用力传感器显示
python scripts/teleop_demo.py --force

# 调整速度（默认 400 mm/s）
python scripts/teleop_demo.py --speed 200
```

## 二、数据采集

### 2.1 采集命令

```bash
# 默认采集: rgbd 模式，3 个 episode，不采集力
python scripts/collect_data.py --dataset datasets/demo.zarr

# 只采集 RGB（不需要 depth）
python scripts/collect_data.py --cam-mode rgb --dataset datasets/rgb_only.zarr

# 指定初始偏移 + episode 数量
python scripts/collect_data.py --start-bias 0 0 -200 --episodes 10 --dataset datasets/plug.zarr

# 启用力传感器
python scripts/collect_data.py --force --dataset datasets/force_demo.zarr

# 保存视频回看（每个 episode 自动生成 arm + fix 两个 MP4）
python scripts/collect_data.py --save-video --dataset datasets/demo.zarr
# → 生成 datasets/demo_ep0_arm.mp4, datasets/demo_ep0_fix.mp4, ...

# 带随机偏移 + 夹爪始终闭合（stamp 类任务）
python scripts/collect_data.py --start-bias 0 0 -270 --gripper-closed \
    --random-bias-x -50 70 --random-bias-y -20 10 \
    --dataset datasets/stamp.zarr

# 组合使用
python scripts/collect_data.py \
    --start-bias 0 0 -200 --episodes 20 --force \
    --cam-mode rgbd --save-video \
    --dataset datasets/plug_force.zarr
```

### 2.2 完整 CLI 参数

**数据集参数:**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--dataset` | `datasets/demo.zarr` | Zarr 数据集路径（不存在则创建，已存在则追加） |
| `--episodes` | `3` | 本次采集的 episode 数量 |

**任务参数（直接通过 CLI 指定，替代旧版 `--task` + YAML 配置）：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--start-bias X Y Z` | `0 0 0` | 相对于 Home 的初始位置偏移 (mm)，3 个值用空格分隔 |
| `--random-bias-x LO HI` | 无 | 每个 episode X 方向随机偏移范围，例如 `--random-bias-x -50 70` |
| `--random-bias-y LO HI` | 无 | 每个 episode Y 方向随机偏移范围，例如 `--random-bias-y -20 10` |
| `--gripper-closed` | 关闭 | 夹爪始终闭合（stamp 类任务） |

**硬件参数:**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--ip` | `192.168.31.232` | XArm IP 地址 |
| `--force` | 关闭 | 启用力传感器（默认不采集力） |

**相机参数:**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--cam-mode` | `rgbd` | 相机模式: `rgb` / `rgbd` / `pcd` |
| `--image-size W H` | `320 240` | 保存图像分辨率（从相机原始分辨率 resize） |

**视频参数:**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--save-video` | 关闭 | 保存每个 episode 的 RGB 视频 |
| `--video-fps` | `15` | 视频帧率 |

**SpaceMouse 参数:**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--trans-scale` | `5.0` | SpaceMouse 平移灵敏度 (mm/tick) |
| `--rot-scale` | `0.004` | SpaceMouse 旋转灵敏度 (rad/tick) |

### 2.3 采集流程详解

```
┌─────────────────────────────────────────────────────┐
│ 1. 启动: 初始化 XArmEnv + 双相机 + SpaceMouse        │
│    ↓                                                 │
│ 2. 复位: 机械臂回 Home → 移动到任务初始位置             │
│    ↓                                                 │
│ 3. 热身: 等待 warmup_time (默认 1s)                   │
│    ↓                                                 │
│ ┌─── Episode 循环 ──────────────────────────────────┐ │
│ │ 4. 【等待】显示提示，等待按空格                      │ │
│ │    - SpaceMouse 左右键可调整夹爪（热身）             │ │
│ │    ↓ 按空格                                        │ │
│ │ 5. 力传感器调零（如果 --force）                      │ │
│ │    ↓                                               │ │
│ │ 6. 【录制中】                                      │ │
│ │    每帧: SpaceMouse → 相机采集 → env.step → 存 buffer │ │
│ │    左键=开夹  右键=关夹                              │ │
│ │    每 50 步打印 FPS + force_z + gripper              │ │
│ │    ↓ 按 Enter                                      │ │
│ │ 7. 保存 buffer → Zarr (+ 释放 VideoWriter)          │ │
│ │    ↓                                               │ │
│ │ 8. 复位 → 移动到任务初始位置 → 下一个 Episode         │ │
│ └────────────────────────────────────────────────────┘ │
│ 9. 计算 episode_ends → 完成                           │
└─────────────────────────────────────────────────────┘
```

### 2.4 任务配置

任务参数通过 CLI 直接指定，无需额外的配置文件。以下是常见任务的对应命令：

**默认任务**（无偏移）:
```bash
python scripts/collect_data.py --dataset datasets/demo.zarr
```

**plug 插入任务**（带随机化）:
```bash
python scripts/collect_data.py \
    --start-bias 0 0 -200 \
    --random-bias-x -50 70 --random-bias-y -20 10 \
    --dataset datasets/plug.zarr
```

**stamp 盖章任务**（夹爪始终闭合）:
```bash
python scripts/collect_data.py \
    --start-bias 0 0 -270 --gripper-closed \
    --dataset datasets/stamp.zarr
```

配置字段说明：

| CLI 参数 | 对应配置 | 说明 |
|------|------|------|
| `--start-bias X Y Z` | `start_bias` | 相对于 Home 的初始偏移 (mm) |
| `--random-bias-x LO HI` | `random_bias.x` | 每个 episode X 方向随机偏移范围（可选） |
| `--random-bias-y LO HI` | `random_bias.y` | 每个 episode Y 方向随机偏移范围（可选） |
| `--gripper-closed` | `gripper_always_closed` | 夹爪始终记为闭合状态（stamp 类任务） |

### 2.5 相机模式

| `--cam-mode` | 存储内容 | 适用场景 |
|:---:|------|---------|
| `rgb` | 仅 RGB 图像 | 纯视觉策略，数据量小 |
| **`rgbd`** (默认) | RGB + depth 分开存储 | **大多数任务推荐** |
| `pcd` | RGB + depth（点云由 RealsenseEnv 计算但不存储） | 需要在线点云的场景 |

### 2.6 夹爪逻辑

| 条件 | gripper_state | gripper_action |
|------|:---:|:---:|
| `gripper_always_closed = true` | 0 (闭合) | 0 (闭合) |
| 夹爪位置 ≤ 420 | 0 (闭合) | 取决于目标 |
| 夹爪位置 > 420 | 1 (张开) | 取决于目标 |
| 目标 = 840 (开) | — | 1 |
| 目标 = 0 (关) | — | 0 |

## 三、Zarr 数据格式

### 3.1 数据集结构

```
dataset.zarr/
├── data/
│   ├── rgb_arm         # (N, 3, 240, 320) uint8   — 臂上相机 RGB, Blosc 压缩
│   ├── rgb_fix         # (N, 3, 240, 320) uint8   — 固定相机 RGB, Blosc 压缩
│   ├── depth_arm       # (N, 1, 240, 320) uint16  — 臂上相机 depth (仅 rgbd/pcd)
│   ├── depth_fix       # (N, 1, 240, 320) uint16  — 固定相机 depth (仅 rgbd/pcd)
│   ├── pos             # (N, 6) float32            — 末端位姿 [x,y,z,r,p,y] (goal_pos)
│   ├── force           # (N, 6) float32            — 6 维外力/力矩 (ext_force, 无力传感器时全 0)
│   ├── action          # (N, 6) float32            — SpaceMouse 原始 6D 动作
│   ├── gripper_state   # (N, 1) float32            — 夹爪状态 (0=闭, 1=开)
│   ├── gripper_action  # (N, 1) float32            — 夹爪目标动作 (0=闭, 1=开)
│   └── episode         # (N,) uint16               — episode 编号
└── meta/
    └── episode_ends    # (M,) uint32               — 各 episode 结束索引（累积）
```

> `--cam-mode rgb` 时不会创建 `depth_arm` / `depth_fix`。

### 3.2 图像格式说明

- **存储格式**: `(N, C, H, W)` — channel-first，与 PyTorch 一致
- **分辨率**: 默认 320×240（从相机原始 640×480 resize）
- **RGB 顺序**: RGB（不是 BGR），直接可用
- **depth**: uint16，单位取决于相机 depth_scale，用 `INTER_NEAREST` 插值保持精度
- **压缩**: Blosc zstd, clevel=3, BITSHUFFLE

### 3.3 读取数据

```python
import zarr
import numpy as np

ds = zarr.open("datasets/demo.zarr", "r")

# 查看基本信息
print("总步数:", ds["data/pos"].shape[0])
print("Episode 数:", ds["meta/episode_ends"].shape[0])
print("Episode 结束索引:", ds["meta/episode_ends"][:])

# 读取第 i 个 episode 的数据
episode_ends = ds["meta/episode_ends"][:]
i = 0
start = 0 if i == 0 else int(episode_ends[i - 1])
end = int(episode_ends[i])

pos = ds["data/pos"][start:end]            # (T, 6) float32
action = ds["data/action"][start:end]      # (T, 6) float32
rgb_arm = ds["data/rgb_arm"][start:end]    # (T, 3, 240, 320) uint8
force = ds["data/force"][start:end]        # (T, 6) float32

# 如果是 rgbd 模式
if "depth_arm" in ds["data"]:
    depth_arm = ds["data/depth_arm"][start:end]  # (T, 1, 240, 320) uint16

# 显示一帧图像
import cv2
frame = rgb_arm[0].transpose(1, 2, 0)  # (3, H, W) → (H, W, 3)
cv2.imshow("arm", cv2.cvtColor(frame, cv2.COLOR_RGB2BGR))
cv2.waitKey(0)
```

### 3.4 增量追加

数据集支持跨次采集追加。如果 `--dataset` 指定的路径已存在：
- 自动检测已有 episode 数量
- 新 episode 从下一个编号开始
- `episode_ends` 在采集结束时自动重新计算

```bash
# 第一次采集: 创建数据集，采集 episode 0~9
python scripts/collect_data.py --episodes 10 --dataset datasets/pick.zarr

# 第二次采集: 追加到同一数据集，采集 episode 10~14
python scripts/collect_data.py --episodes 5 --dataset datasets/pick.zarr
```

## 四、视频保存

开启 `--save-video` 后，每个 episode 录制期间会同时保存两个视角的 MP4 视频：

```
datasets/
├── demo.zarr/          # Zarr 数据集
├── demo_ep0_arm.mp4    # Episode 0 臂上相机
├── demo_ep0_fix.mp4    # Episode 0 固定相机
├── demo_ep1_arm.mp4    # Episode 1 臂上相机
├── demo_ep1_fix.mp4    # Episode 1 固定相机
└── ...
```

- 编码: mp4v
- 分辨率: 与采集图像一致 (默认 320×240)
- 帧率: `--video-fps` 指定 (默认 15)
- 用途: 快速回看采集质量，不影响 Zarr 数据

## 五、注意事项

### 安全

- **运动速度**: 采集时 `env.step()` 默认 speed=100 mm/s（Collector 内部写死），比较保守。遥操作脚本 `teleop_demo.py` 默认 speed=400 mm/s，可通过 `--speed` 调整
- 随时准备按 **Ctrl+C** 中断采集
- 机械臂底座有**硬件急停按钮**，紧急情况直接拍

### 采集质量

- **力传感器调零**: 每个 episode 开始录制前自动调零（需 `--force`）
- **采集频率**: 目前无固定频率控制，实际 FPS 取决于硬件响应（通常 12~20 Hz）
- 确保遥操作动作流畅，避免大幅度抖动
- 关注终端输出的 `force_z`，接触力异常大说明可能卡住
- 每 50 步打印 FPS，低于 10 说明有问题（检查相机连接）

### 常见问题

| 问题 | 排查 |
|------|------|
| SpaceMouse 无反应 | `lsusb` 检查 USB 连接；确认 `hidapi` 已安装 |
| 相机画面黑屏 | 先跑 `python scripts/test_cameras.py` 单独验证 |
| 采集 FPS 很低 (<10) | 检查 USB 带宽（双相机共用一个 USB Hub 会互抢） |
| Zarr 追加时 shape 不匹配 | 确认 `--cam-mode` 和上次采集一致 |
| 视频文件 0 字节 | Episode 录制了 0 步（可能没按空格就按了 Enter） |
