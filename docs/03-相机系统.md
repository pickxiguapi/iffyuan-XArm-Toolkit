# 相机系统

> 这篇解决的问题：怎么初始化双相机、RGB/RGBD/点云怎么选、observation 里有什么。

## 基本用法

```python
from xarm_toolkit.env import RealsenseEnv

# 打开臂上 D435i（只要 RGB）
cam_arm = RealsenseEnv(serial="327122075644", mode="rgb")

# 打开固定 L515（要 RGB + Depth）
cam_fix = RealsenseEnv(serial="f1271506", mode="rgbd")

# 拍一帧
obs = cam_arm.step()
```

## 初始化参数

```python
RealsenseEnv(
    serial="327122075644",   # 相机序列号（必填）
    mode="rgbd",              # 观测模式
    record=False,             # 是否录制 .bag 文件（调试用）
)
```

| 参数 | 说明 |
|------|------|
| `serial` | **必填**。用错了会打开别的相机，用 `list_devices()` 确认 |
| `mode` | `"rgb"` / `"rgbd"` / `"pcd"`，见下面详解 |
| `record` | `True` 时录制到 `debug.bag`，调试相机问题时用 |

## 三种观测模式

| 模式 | 返回内容 | 适用场景 |
|------|---------|---------|
| `"rgb"` | 只有彩色图 | 数据采集（只需要 RGB） |
| `"rgbd"` | 彩色图 + 深度图 | 需要深度信息的任务 |
| `"pcd"` | RGBD + 点云 | 需要 3D 点云的任务 |

### Observation 字典

```python
# mode="rgb"
obs = {
    "rgb": o3d.t.geometry.Image,       # 彩色图 (H, W, 3)
    "intrinsic_matrix": np.ndarray,     # 相机内参 (3, 3)
    "depth_scale": float,               # 深度缩放因子
}

# mode="rgbd"
obs = {
    "rgbd": o3d.t.geometry.RGBDImage,  # 彩色 + 深度
    "intrinsic_matrix": np.ndarray,
    "depth_scale": float,
}

# mode="pcd"
obs = {
    "rgbd": o3d.t.geometry.RGBDImage,
    "pcd": o3d.t.geometry.PointCloud,  # 点云
    "intrinsic_matrix": np.ndarray,
    "depth_scale": float,
}
```

### 转成 numpy 用

```python
import numpy as np

obs = cam_arm.step()

# RGB 模式
rgb = np.asarray(obs["rgb"])  # (480, 640, 3) uint8, RGB 顺序

# RGBD 模式
color = np.asarray(obs["rgbd"].color)  # (480, 640, 3) uint8
depth = np.asarray(obs["rgbd"].depth)  # (480, 640, 1) uint16

# 用 OpenCV 显示（需要 RGB → BGR）
import cv2
cv2.imshow("arm", cv2.cvtColor(rgb, cv2.COLOR_RGB2BGR))
cv2.waitKey(1)
```

## 双相机配置

我们使用两个相机，各有不同的硬件配置：

| 相机 | 型号 | Serial | 分辨率 | FPS | 特殊配置 |
|------|------|--------|--------|-----|---------|
| 臂上 | D435i | `327122075644` | 640x480 RGB + 640x480 Depth | 30 | 无 |
| 固定 | L515 | `f1271506` | 640x480 RGB + 640x480 Depth | 30 | `MAX_RANGE` 模式 |

配置写在 `realsense_env.py` 的 `CAMERA_CONFIGS` 字典里。如果换了新相机，在那里加一条就行。

## 双相机同时使用

```python
import cv2
import numpy as np
from xarm_toolkit.env import RealsenseEnv

# 分别初始化
cam_arm = RealsenseEnv(serial="327122075644", mode="rgb")
cam_fix = RealsenseEnv(serial="f1271506", mode="rgb")

# 采集循环
while True:
    arm_obs = cam_arm.step()
    fix_obs = cam_fix.step()

    arm_rgb = np.asarray(arm_obs["rgb"])
    fix_rgb = np.asarray(fix_obs["rgb"])

    cv2.imshow("ARM (D435i)", cv2.cvtColor(arm_rgb, cv2.COLOR_RGB2BGR))
    cv2.imshow("FIX (L515)", cv2.cvtColor(fix_rgb, cv2.COLOR_RGB2BGR))

    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cv2.destroyAllWindows()
```

> **注意**：两个相机的 `step()` 不是严格同步的，会有几毫秒的时间差。对数据采集来说通常没问题（30fps 下每帧 33ms）。

## 数据采集时的图像处理

采集存 Zarr 时，图像会做如下处理：

```python
# 原始 640x480 → 缩放到 320x240
rgb_resized = cv2.resize(rgb, (320, 240))

# HWC → CHW（PyTorch 格式）
rgb_chw = rgb_resized.transpose(2, 0, 1)  # (3, 240, 320)
```

## 常见问题

### 相机初始化失败

```
RuntimeError: RealSense init failed
```

排查步骤：
1. 确认 USB 3.0 口（USB 2.0 不行）
2. 确认 serial 号对不对：`o3d.t.io.RealSenseSensor.list_devices()`
3. 拔插 USB 重试
4. 确认没有其他程序占用相机（realsense-viewer、其他 Python 进程）

### 只有一个相机能用

两个相机对 USB 带宽要求较高。如果插在同一个 USB Hub 上可能带宽不够。解决：
- 分别插在主板不同的 USB 控制器上
- 降低分辨率或 FPS

### 画面有延迟

Open3D 的 RealSenseSensor 内部有小缓冲区，延迟约 1~2 帧（30~60ms），属于正常范围。
