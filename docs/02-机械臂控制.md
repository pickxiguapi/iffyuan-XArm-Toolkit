# 机械臂控制

> 这篇解决的问题：怎么用 `XArmEnv` 控制机械臂，三种模式怎么选，速度/力控怎么调。

## 基本用法

```python
from xarm_toolkit.env import XArmEnv

env = XArmEnv(
    addr="192.168.31.232",
    use_force=False,
    action_mode="delta_eef",
    initial_gripper_position=840, # 开夹爪
)

obs = env.reset()       # 复位到 Home
obs = env.step(action, gripper_action=None)  # 执行一步
env.cleanup()           # 断开连接（也会被 atexit 自动调用）
```

## 初始化参数详解

```python
XArmEnv(
    addr="192.168.31.232",       # 机械臂 IP
    use_force=False,              # 是否启用力传感器
    action_mode="delta_eef",      # 控制模式
    initial_gripper_position=840, # 启动时夹爪位置
    max_retries=10,               # SDK 调用失败的最大重试次数
)
```

| 参数 | 默认值 | 什么时候需要改 |
|------|--------|---------------|
| `addr` | `"192.168.31.232"` | 机械臂 IP 变了才改 |
| `use_force` | `False` | **数据采集时改为 `True`**（需要力传感器数据）；纯位控不需要 |
| `action_mode` | `"delta_eef"` | 不同的动作空间和控制模式，见下面详解 |
| `initial_gripper_position` | `840` | 840 = 全开，0 = 闭合。启动时想夹住东西就改成 0 |
| `max_retries` | `10` | 一般不用改。网络不稳定可以调大 |

## 三种控制模式

### 1. `delta_eef` — 增量末端控制（遥操作用这个）

每一步发送 **相对于当前位姿的增量**：

```python
env = XArmEnv(action_mode="delta_eef")
obs = env.reset()

# 向 X+ 方向移动 5mm
obs = env.step(
    action=[5, 0, 0, 0, 0, 0],   # [dx, dy, dz, droll, dpitch, dyaw]
    speed=1000,                    # 笛卡尔速度，0~2000
)
```

- `action` 前三维：mm 单位的位移增量
- `action` 后三维：rad 单位的旋转增量
- 内部维护 `goal_pos`，每步 `goal_pos += action`

> **SpaceMouse 遥操作**就是用这个模式，每个控制周期发送 SpaceMouse 读数作为 delta。

### 2. `absolute_eef` — 绝对末端控制（VLA 笛卡尔部署）

每一步发送 **目标绝对位姿**：

```python
env = XArmEnv(action_mode="absolute_eef")
obs = env.reset()

# 移动到指定位姿
target = obs["cart_pos"].copy()
target[2] += 50   # z 方向抬高 50mm
obs = env.step(
    action=target,               # [x, y, z, roll, pitch, yaw]
    speed=1000,
)
```

- `action` 是 6D 绝对位姿，xyz 单位 mm，rpy 单位 rad

### 3. `absolute_joint` — 绝对关节控制（Pi0 关节部署）

每一步发送 **目标关节角度**：

```python
env = XArmEnv(action_mode="absolute_joint")
obs = env.reset()

# 移动到指定关节角
target_joints = obs["servo_angle"].copy()
target_joints[1] += 0.1   # 第 2 个关节偏移 0.1 rad
obs = env.step(
    action=target_joints,        # [j1, j2, j3, j4, j5, j6] (rad)
    joint_speed=0.5,             # 关节角速度 rad/s
)
```

- `action` 是 6 个关节角，单位 rad
- `joint_speed` 单位 rad/s（1.0 ≈ 57 度/秒）
- 适合 Pi0 等直接输出关节角的 VLA 模型

## `step()` 参数详解

```python
obs = env.step(
    action,                # 动作（含义取决于 action_mode）
    gripper_action=None,   # 夹爪目标位置，None 表示不动
    speed=1000,            # 笛卡尔速度（仅 eef 模式）
    joint_speed=1.0,       # 关节角速度（仅 joint 模式）
)
```

| 参数 | 范围 | 说明 |
|------|------|------|
| `gripper_action` | `0` ~ `840` 或 `None` | 0=闭合，840=全开。传 `None` 跳过夹爪控制 |
| `speed` | `0` ~ `2000` | 笛卡尔运动速度。遥操作建议 100~300（安全），部署**确定安全**可以 500~1000 |
| `joint_speed` | `> 0` (rad/s) | 关节运动速度。0.5 比较稳，1.0 较快 |

> **安全提示**：第一次调试时 `speed` 设小一点（100~300），确认方向对了再加速。

## `reset()` 详解

```python
obs = env.reset(close_gripper=True)
```

复位流程：
1. 夹爪动作：`close_gripper=True` 先闭合（避免甩出夹持物），`False` 打开
2. 如果开了力传感器，先禁用（避免复位运动触发力保护）
3. 切换到关节控制模式（Mode 6），关节空间回到 Home
4. Home 位姿：`[470, 0, 530, π, 0, -π/2]` (mm/rad)
5. Home 关节角：`[0, 0, -π/2, 0, π/2, π/2, 0]` (rad)
6. 等待到达（位置误差 < 20mm 且旋转误差 < 0.02 rad）
7. 恢复原控制模式，重新启用力传感器

## 力传感器

### 力传感器使用

```python
env = XArmEnv(use_force=True)
obs = env.reset()

# obs 中的力数据
print(obs["ext_force"])   # 外部力 [fx, fy, fz, tx, ty, tz]  # 一般只要看这个数据
print(obs["raw_force"])   # 原始力传感器读数  # 这数据没啥用 

# 采集前重新调零（推荐每个 episode 开始前调一次）
success = env.reset_force_sensor_zero()
```

> **注意**：力传感器在机械臂运动过程中数据会漂，每个采集 episode 开始前调用 `reset_force_sensor_zero()` 可以消除零点漂移。

## Observation 字典

`reset()` 和 `step()` 都返回相同结构的 obs：

```python
obs = {
    "cart_pos":         np.ndarray,  # (6,) 末端位姿 [x,y,z,roll,pitch,yaw] mm/rad
    "servo_angle":      np.ndarray,  # (6,) 关节角 (rad)，只取前 6 个
    "ext_force":        np.ndarray,  # (6,) 外部力，use_force=False 时为 None
    "raw_force":        np.ndarray,  # (6,) 原始力，use_force=False 时为 None
    "goal_pos":         np.ndarray,  # (6,) 当前目标位姿
    "gripper_position": np.ndarray,  # 夹爪位置标量
}
```

**`cart_pos` vs `goal_pos` 的区别**：
- `cart_pos`：机械臂**实际**到达的位姿（从编码器读取）
- `goal_pos`：代码**下发**的目标位姿
- 两者之间可能有微小误差（1~2mm），高速运动时差距更大

## SDK Mode 说明

XArmEnv 内部使用了两种 SDK Mode：

| Mode | 用途 | 什么时候激活 |
|------|------|-------------|
| Mode 7 | 笛卡尔在线规划（低延迟） | `delta_eef` / `absolute_eef` 正常运行 |
| Mode 6 | 关节在线规划 | `absolute_joint` 运行 / `reset()` 回 Home |

一般不需要手动管理 Mode，`XArmEnv` 内部会自动切换。

## 完整示例：三种模式对比

```python
import time
from xarm_toolkit.env import XArmEnv

# --- 示例 1: delta_eef（遥操作场景） ---
env = XArmEnv(action_mode="delta_eef", use_force=False)
obs = env.reset()
for _ in range(50):
    obs = env.step([0, 0, 0, 0, 0, 0], speed=1000)
    time.sleep(0.02)

# --- 示例 2: absolute_eef（VLA 笛卡尔部署） ---
env2 = XArmEnv(action_mode="absolute_eef", use_force=False)
obs = env2.reset()
target = obs["cart_pos"].copy()
target[2] += 50
for _ in range(5):
    obs = env2.step(target, speed=1000)
    time.sleep(0.02)

# --- 示例 3: absolute_joint（Pi0 关节部署） ---
env3 = XArmEnv(action_mode="absolute_joint", use_force=False)
obs = env3.reset()
target_j = obs["servo_angle"].copy()
target_j[1] += 0.1
for _ in range(5):
    obs = env3.step(target_j, joint_speed=0.5)
    time.sleep(0.02)
```
