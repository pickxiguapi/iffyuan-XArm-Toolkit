### 3.3 数据特征映射

无论哪种方案，最终的 LeRobot v3 数据集特征一致：

| LeRobot 特征 | shape | 来源 | 说明 |
|-------------|-------|------|------|
| `observation.image` | (240, 320, 3) uint8 | 固定相机 L515 | 主视角 RGB |
| `observation.wrist_image` | (240, 320, 3) uint8 | 臂上相机 D435i | 手腕视角 RGB |
| `observation.state` | (7,) float32 | pos(6) + gripper_state(1) | 末端位姿 [x,y,z,r,p,y] + 夹爪 (0/1) |
| `action` | (7,) float32 | action(6) + gripper_action(1) | delta EEF [dx,dy,dz,dr,dp,dy] + 夹爪 (0/1) |

#### Zarr → LeRobot 字段映射详情

```
Zarr 原始字段                       LeRobot v3 特征                       拼接方式
──────────────────────────────────  ──────────────────────────────────── ──────────────────
data/rgb_fix  (N, 3, 240, 320)  →  observation.image      (240,320,3)   CHW→HWC, 嵌入 parquet
data/rgb_arm  (N, 3, 240, 320)  →  observation.wrist_image (240,320,3)  CHW→HWC, 嵌入 parquet
data/pos      (N, 6)            ┐
data/gripper_state (N, 1)       ┘→ observation.state      (7,)          concat([pos, gripper_state])
data/action   (N, 6)            ┐
data/gripper_action (N, 1)      ┘→ action                 (7,)          concat([action, gripper_action])
meta/episode_ends               →  episodes 元数据（LeRobot 自动管理）
                                →  task: str                             --task 参数传入
```

> **未转入的字段**：`data/force (N,6)`、`data/depth_arm`、`data/depth_fix` 未转入 LeRobot，ACT/Pi0.5 当前只用 RGB + state + action。如需力控或深度训练，需修改 `convert_to_lerobot.py` 添加对应 feature。

#### observation.state 各维度含义

| 索引 | 名称 | 单位 | 来源 | 典型范围 |
|------|------|------|------|----------|
| 0 | x | mm | `env.goal_pos[0]` 末端 X | 150 ~ 450 |
| 1 | y | mm | `env.goal_pos[1]` 末端 Y | -300 ~ 300 |
| 2 | z | mm | `env.goal_pos[2]` 末端 Z | 50 ~ 500 |
| 3 | roll | rad | `env.goal_pos[3]` 末端 Roll | -π ~ π |
| 4 | pitch | rad | `env.goal_pos[4]` 末端 Pitch | -π ~ π |
| 5 | yaw | rad | `env.goal_pos[5]` 末端 Yaw | -π ~ π |
| 6 | gripper | 0 or 1 | 夹爪状态 | 0=张开, 1=闭合 |

#### action 各维度含义

| 索引 | 名称 | 单位 | 来源 | 说明 |
|------|------|------|------|------|
| 0 | dx | mm | SpaceMouse delta X | 单步位移增量 |
| 1 | dy | mm | SpaceMouse delta Y | 单步位移增量 |
| 2 | dz | mm | SpaceMouse delta Z | 单步位移增量 |
| 3 | droll | rad | SpaceMouse delta Roll | 单步旋转增量 |
| 4 | dpitch | rad | SpaceMouse delta Pitch | 单步旋转增量 |
| 5 | dyaw | rad | SpaceMouse delta Yaw | 单步旋转增量 |
| 6 | gripper | 0 or 1 | 夹爪动作 | 0=张开, 1=闭合 |

#### 转换后磁盘文件结构

LeRobot v3（`use_videos=False`）将图像以二进制 blob 嵌入 parquet，不单独存 PNG：

```
lerobot_datasets/act_pick2/                  # --output 路径
├── meta/
│   ├── info.json                            # fps, robot_type, repo_id, features 定义
│   ├── tasks.parquet                        # task_index → task 描述映射
│   ├── stats.json                           # 各特征的归一化统计量 (训练自动读取)
│   └── episodes/
│       └── chunk-000/
│           └── file-000.parquet             # 各 episode 的元信息
├── data/
│   └── chunk-000/
│       ├── file-000.parquet                 # ~80-120MB 每个
│       ├── file-001.parquet                 # 包含: state(7) + action(7) + 双相机图像(blob)
│       ├── ...                              #       + timestamp + episode_index + task_index
│       └── file-029.parquet                 # 每个 episode 一个 file
└── images/                                  # 空目录 (图像已嵌入 parquet，此处不存文件)
    ├── observation.image/
    └── observation.wrist_image/
```

> **注意**：`images/` 目录下是空的，这是正常的。LeRobot v3 默认把图像嵌入 parquet 文件，所以每个 parquet 文件有 80~120MB（含图像二进制数据）。

#### 转换参数说明

| 参数 | 当前值 | 说明 |
|------|--------|------|
| `use_videos=False` | 图像嵌入 parquet | 无损、加载快，但磁盘占用较大 |
| `image_writer_threads=4` | 4 线程并行写图 | 加速转换 |
| `fps` | 由 `--fps` 指定 | 训练时 temporal 采样基准，建议与部署推理频率一致 |
| `robot_type` | `xarm6` | 写入 info.json，训练配置需匹配 |

## 五、关键参数速查

### 5.1 ACT 模型参数

| 参数 | 默认值 | 说明 | 调参建议 |
|------|--------|------|----------|
| **chunk_size** | 100 | 模型一次预测多少步动作 | 低频任务(10Hz) 用 30~50，高频(30Hz) 用 50~100 |
| **n_action_steps** | 100 | 每次策略调用实际执行的步数 | 部署时建议 1（每步推理），训练时等于 chunk_size |
| **vision_backbone** | `resnet18` | 图像编码器 | resnet18 够用，数据多可试 resnet34 |
| **pretrained_backbone_weights** | `ResNet18_Weights.IMAGENET1K_V1` | 预训练权重 | 建议保留，加速收敛 |
| **dim_model** | 512 | Transformer 隐层维度 | 小数据集可降到 256 省显存 |
| **n_heads** | 8 | 注意力头数 | 默认即可 |
| **dim_feedforward** | 3200 | FFN 中间维度 | 默认即可 |
| **n_encoder_layers** | 4 | 编码器层数 | 默认即可 |
| **n_decoder_layers** | 1 | 解码器层数 | 原 ACT 实现只用 1 层（已知 bug，见下文） |
| **use_vae** | True | 是否使用 VAE 变分目标 | 建议开启，学习多模态动作分布 |
| **latent_dim** | 32 | VAE 潜在空间维度 | 默认即可 |
| **n_vae_encoder_layers** | 4 | VAE 编码器层数 | 默认即可 |
| **kl_weight** | 10.0 | KL 散度损失权重 | 训练不稳定时可降到 1.0~5.0 |
| **dropout** | 0.1 | Transformer dropout | 数据少可提高到 0.2 防过拟合 |
| **temporal_ensemble_coeff** | None | 时间集成系数（推理时使用） | 部署时设 0.01 可平滑动作，需 n_action_steps=1 |

### 5.2 训练参数

| 参数 | 默认值 | 说明 | 调参建议 |
|------|--------|------|----------|
| **batch_size** | 8 | 训练批大小 | 显存够就 32~64，不够就 8~16 |
| **training_steps** | 100,000 | 总训练步数 | 50 episode → 50k，200+ episode → 100k~200k |
| **optimizer_lr** | 1e-5 | 学习率 | ACT 默认 1e-5，一般不用改 |
| **optimizer_lr_backbone** | 1e-5 | 骨干网络学习率（和主学习率一致） | 默认即可 |
| **optimizer_weight_decay** | 1e-4 | 权重衰减 | 默认即可 |
| **num_workers** | 4 | DataLoader 工作进程数 | 默认即可 |
| **fps** | — | 数据集帧率 | 转换/采集时设定，影响 chunk 覆盖的时间窗口 |

### 5.3 chunk_size 与 fps 的关系

`chunk_size` 决定模型预测的时间窗口长度：

```
时间窗口 = chunk_size / fps

例：
  chunk_size=50, fps=10  → 预测未来 5.0 秒
  chunk_size=50, fps=15  → 预测未来 3.3 秒
  chunk_size=100, fps=30 → 预测未来 3.3 秒
```

一般操作任务预测 1 秒即可。chunk_size 太大会增加显存，太小会导致动作不连贯。

## 六、ACT 模型架构

```
输入观察 (双相机 RGB + 7D state)
        ↓
   ResNet18 视觉骨干（每张图独立编码）
        ↓
   位置编码 + 展平为 token 序列
        ↓
VAE 编码器（仅训练时，输入: CLS + state + GT 动作序列）
        ↓
   采样潜在变量 z ~ N(μ, σ²)
        ↓
   Transformer 编码器 (4 层)
   输入 tokens: [z, state_embed, img_tokens...]
        ↓
   Transformer 解码器 (1 层)
   输入: chunk_size 个可学习 query
   交叉注意力 → 编码器输出
        ↓
   线性动作头
        ↓
输出: (batch, chunk_size, 7) — 7D 动作序列
```

### 损失函数

```
总损失 = L1_loss + kl_weight × KLD_loss

L1_loss:  预测动作与真实动作的平均绝对误差（逐元素）
KLD_loss: D_KL(N(μ, σ²) || N(0, I))，仅 use_vae=True 时计算
```

- 训练时 VAE 编码器能看到 ground-truth 动作，采样 z 作为条件
- 推理时没有 GT 动作，z 直接设为全零（即标准正态的均值）

## 七、归一化

ACT 默认对所有特征使用 **MEAN_STD**（均值方差标准化）归一化：

```python
normalization_mapping = {
    "VISUAL": NormalizationMode.MEAN_STD,   # 图像
    "STATE":  NormalizationMode.MEAN_STD,   # 机器人状态
    "ACTION": NormalizationMode.MEAN_STD,   # 动作
}
```

- **训练时**：自动从数据集 `stats` 读取均值/方差，通过 `preprocessor` 归一化输入
- **保存时**：归一化统计量随 checkpoint 一起保存（`preprocessor.save_pretrained()`）
- **推理时**：通过 `make_pre_post_processors` 从 checkpoint 加载，无需手动处理

## 八、推理与部署

### 8.1 加载 Checkpoint

```python
import torch
from lerobot.policies.act.modeling_act import ACTPolicy
from lerobot.datasets.dataset_metadata import LeRobotDatasetMetadata
from lerobot.policies.factory import make_pre_post_processors

device = torch.device("cuda")

# 加载模型
model = ACTPolicy.from_pretrained("outputs/act_xarm6")
model.eval()
model.to(device)

# 加载归一化统计量
dataset_metadata = LeRobotDatasetMetadata("xarm6_demo", root="lerobot_datasets/demo")
preprocess, postprocess = make_pre_post_processors(
    model.config, dataset_stats=dataset_metadata.stats
)
```

### 8.2 推理循环

```python
# 每个新 episode 开始前必须 reset（清空动作 chunk 队列）
model.reset()

for step in range(max_steps):
    # 1. 获取观测（需要匹配训练时的特征 key 和 shape）
    obs_dict = {
        "observation.image": fixed_cam_rgb,          # (3, 240, 320) float tensor
        "observation.wrist_image": wrist_cam_rgb,    # (3, 240, 320) float tensor
        "observation.state": state_7d,               # (7,) float tensor
    }

    # 2. 预处理（归一化）
    obs = preprocess(obs_dict)

    # 3. 获取动作
    action = model.select_action(obs)

    # 4. 后处理（反归一化）
    action = postprocess(action)

    # 5. 执行
    # action 是 (7,) tensor: delta_eef(6) + gripper_action(1)
    robot.step(action[:6], action[6])
```

### 8.3 动作 Chunk 队列机制

`select_action()` 内部自动管理动作队列：

1. **首次调用** → 模型推理，预测 `chunk_size` 步动作，缓存到队列
2. **后续调用** → 从队列弹出下一个动作，无需重新推理
3. **队列耗尽**（弹出 `n_action_steps` 个后）→ 重新推理补充队列

如果 `n_action_steps=1`：每步都重新推理（适合开启 temporal ensemble）。
如果 `n_action_steps=chunk_size`：一次推理执行完所有步。

### 8.4 Temporal Ensemble（时间集成）

```python
# 在 ACTConfig 中设置
cfg = ACTConfig(
    temporal_ensemble_coeff=0.01,  # 启用时间集成
    n_action_steps=1,              # 必须设为 1
    ...
)
```

开启后，每步都会推理一次 chunk，然后对历史多次预测的同一时间步做指数加权平均：

```
权重 wᵢ = exp(-coeff × i)，其中 i=0 是最旧的预测
```

效果是平滑动作输出，减少抖动。适合对平滑性要求高的任务。

## 九、与 XArm Toolkit 部署模块集成

ACT 推理可以集成到现有的 VLA Server/Client 架构中：

```python
# GPU 端 — VLAServer 包装 ACT
from xarm_toolkit.deploy.server import VLAServer

class ACTPolicy:
    def __init__(self, model, preprocess, postprocess, device):
        self.model = model
        self.preprocess = preprocess
        self.postprocess = postprocess
        self.device = device

    def predict(self, images, instruction, state):
        obs = {
            "observation.image": images["fixed"],
            "observation.wrist_image": images["wrist"],
            "observation.state": state,
        }
        obs = self.preprocess(obs)
        action = self.model.select_action(obs)
        action = self.postprocess(action)
        return {"actions": action.numpy()}

server = VLAServer(policy=act_policy, host="0.0.0.0", port=8765)
server.run()

# 机器人端 — VLAClient 照常使用
from xarm_toolkit.deploy.client import VLAClient
client = VLAClient(host="gpu-server", port=8765)
result = client.predict(images, instruction, state)
```

## 十一、常见问题

| 问题 | 排查 |
|------|------|
| `CUDA out of memory` | 降低 `batch_size`（8→4）或 `dim_model`（512→256） |
| loss 不下降 | 检查数据集是否正确（`verify_lerobot.py`），确认归一化正常 |
| loss 中 KLD 很大 | 降低 `kl_weight`（10→1），或检查数据多样性 |
| 推理动作抖动 | 开启 `temporal_ensemble_coeff=0.01`，设 `n_action_steps=1` |
| 推理动作不动 | 检查归一化统计量是否匹配（训练和推理用同一数据集 stats） |
| `n_obs_steps != 1` 报错 | 当前 ACT 实现只支持 `n_obs_steps=1`，不要改这个值 |
| `n_action_steps > chunk_size` 报错 | `n_action_steps` 不能大于 `chunk_size` |
| 数据集加载慢 | 增加 `num_workers`；使用 `use_videos=False`（图像而非 MP4） |
| `ModuleNotFoundError: lerobot` | `cd lerobot && pip install -e .` |
