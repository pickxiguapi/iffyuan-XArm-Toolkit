# 模型部署

> 这篇解决的问题：怎么把训练好的 Pi0.5 VLA 模型跑到 XArm6 真机上。

## 部署架构

部署采用 **Server-Client 分离架构**，GPU 推理和机器人控制分别运行在两台机器上，通过 WebSocket + msgpack 通信：

```
┌──────────────────────────────────────┐     ┌──────────────────────────────────────┐
│          GPU 机器 (Server)            │     │         机器人机器 (Client)            │
│                                      │     │                                      │
│  openpi config (.py)                 │     │  XArmEnv (delta_eef)                 │
│       ↓                              │     │  RealsenseEnv × 2 (D435i + L515)     │
│  Pi05XArmPolicy                      │     │       ↓                              │
│    · load_config_from_file()         │     │  build_state():                      │
│    · policy.infer(obs) → actions     │     │    goal_pos(6) + gripper_binary(1)   │
│       ↓                              │     │       ↓                              │
│  VLAServer (WebSocket :10093)  ◄─────┼─────┤  VLAClient ──→ predict()            │
│    · 接收 images + state + prompt    │     │       ↓                              │
│    · 返回 actions (chunk_size, 7)    │     │  XArmDeployer.run():                 │
│                                      │     │    actions[0][:6] → env.step()       │
│  scripts/openpi_server.py            │     │    actions[0][6]  → gripper 0/840    │
└──────────────────────────────────────┘     │                                      │
                                             │  scripts/deploy_vla.py               │
                                             └──────────────────────────────────────┘
```

## 数据格式约定

部署时的数据格式与采集/训练完全一致：

| 字段 | 格式 | 说明 |
|------|------|------|
| `observation/image` | HWC uint8 | 固定相机 (L515) RGB |
| `observation/wrist_image` | HWC uint8 | 臂上相机 (D435i) RGB |
| `observation/state` | float32 (7,) | `goal_pos(6) + gripper(1)` |
| `prompt` | str | 语言指令 |
| **输出** actions | float32 (chunk_size, 7) | `delta_eef(6) + gripper(1)` |

**State 细节：**
- `goal_pos`: commanded 目标位姿 `[x, y, z, roll, pitch, yaw]`，与 Collector 存的 `pos` 一致
- `gripper`: 二值 0/1（≤420 → 0 闭合, >420 → 1 张开），与 Collector 的 `gripper_state` 一致

**Action 细节：**
- 前 6 维: delta EEF `[dx, dy, dz, dr, dp, dy]`
- 第 7 维: gripper 二值（模型输出 ≥0.5 → 840 张开, <0.5 → 0 闭合）

## 前置准备

### Config 文件

openpi 训练时会产生一个 config Python 文件，其中必须定义 `build_config()` 函数，返回 openpi `TrainConfig`。部署时直接指定该文件路径。

示例 config 文件结构：

```python
# pi05_xarm_v2_stage1.py
def build_config():
    """返回 openpi TrainConfig."""
    from openpi.training.config import TrainConfig
    # ... 构建并返回 config
    return config
```

例如在本例子中，这个文件位于:`/home/ubuntu/openpi/pi05_xarm_v2_stage1.py`

### Checkpoint

训练完成后的 checkpoint 目录，例如：

```
/home/ubuntu/models/xarm6_act_pick2_pi05
```

## 启动步骤

### Step 1: 启动 GPU 端推理服务

在 GPU 机器上运行：

```bash
python scripts/openpi_server.py \
    --config-file /home/ubuntu/demo/openpi05/config.py \
    --ckpt-dir /home/ubuntu/models/xarm6_act_pick2_pi05 \
    --port 10093
```

**参数说明：**

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `--config-file` | ✅ | — | openpi config Python 文件路径 |
| `--ckpt-dir` | ✅ | — | checkpoint 目录路径 |
| `--port` | | 10093 | WebSocket 服务端口 |
| `--host` | | 0.0.0.0 | 绑定地址 |
| `--idle-timeout` | | 0 | 连接空闲超时秒数（0=不超时） |

启动成功后会看到：

```
[INFO] Loaded config from: /path/to/pi05_xarm_v2_stage1.py
[INFO] Using local checkpoint: /path/to/checkpoint/29999
[INFO] Creating trained policy ...
[INFO] Pi0.5 XArm policy ready.
[INFO] Starting OpenPI server on 0.0.0.0:10093
```

### Step 2: 启动机器人端部署

在机器人机器上运行（确保 XArm 和相机已连接）：

```bash
python scripts/deploy_vla.py \
    --server-host <GPU机器IP> \
    --instruction "pick up the orange toy and place it to the plate" \
    --hz 10
```

**参数说明：**

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `--server-host` | | localhost | GPU 端 server IP |
| `--server-port` | | 10093 | GPU 端 server 端口 |
| `--instruction` | ✅ | — | 语言指令 |
| `--ip` | | 192.168.31.232 | XArm 控制器 IP |
| `--action-mode` | | delta_eef | 动作模式 |
| `--max-steps` | | 500 | 最大执行步数 |
| `--hz` | | 10.0 | 控制频率 (Hz) |
| `--use-force` | | false | 是否开启力控 |

### 完整示例（两台机器）

```bash
# ===== GPU 机器 (192.168.1.100) =====
python scripts/openpi_server.py \
    --config-file /home/ubuntu/openpi/config/pi05_xarm_v2_stage1.py \
    --ckpt-dir /home/ubuntu/openpi/checkpoints/pi05_xarm_v2_stage1/stage1_30k/29999 \
    --port 10093

# ===== 机器人机器 =====
python scripts/deploy_vla.py \
    --server-host 192.168.1.100 \
    --instruction "pick up the orange toy and place it to the plate" \
    --hz 10 --max-steps 800
```

## Python API 用法

如果需要在代码中直接调用（不走 scripts 入口）：

### Server 端

```python
from xarm_toolkit.deploy.openpi_server import Pi05XArmPolicy
from xarm_toolkit.deploy.server import VLAServer

policy = Pi05XArmPolicy(
    config_file="/path/to/pi05_xarm_v2_stage1.py",
    ckpt_dir="/path/to/checkpoint/29999",
)

server = VLAServer(
    policy=policy,
    port=10093,
    metadata={
        "model": "pi0.5",
        "action_format": "delta_eef(6)+gripper(1)",
    },
)
server.run()  # 阻塞
```

### Client 端

```python
from xarm_toolkit.deploy.robot_deploy import XArmDeployer

deployer = XArmDeployer(
    server_host="192.168.1.100",
    server_port=10093,
    instruction="pick up the orange toy and place it to the plate",
    arm_ip="192.168.31.232",
    action_mode="delta_eef",
    max_steps=300,
    hz=10.0,
)
deployer.run()  # 阻塞，Ctrl+C 退出
```

### 单独使用 VLAClient（自定义控制循环）

```python
from xarm_toolkit.deploy.client import VLAClient
import numpy as np

client = VLAClient(host="192.168.1.100", port=10093)

# 发送推理请求
result = client.predict(
    images=[rgb_arm, rgb_fix],          # list[np.ndarray], HWC uint8
    instruction="pick up the cup",
    state=np.array([470, 0, 530, 3.14, 0, -1.57, 1], dtype=np.float32),
)

actions = result["actions"]  # (chunk_size, 7)
print(actions.shape)         # e.g. (50, 7)

client.close()
```

## 部署控制循环详解

`XArmDeployer.run()` 的每一步执行：

```
1. 相机采集 rgb_arm (wrist), rgb_fix (base)
2. 读取 obs → build_state(): goal_pos(6) + gripper_binary(1)
3. VLAClient.predict() → server 推理 → actions (chunk_size, 7)
4. 取 actions[0]:
   - [:6] → env.step(action=delta_eef)
   - [6]  → gripper: ≥0.5 → 840(开), <0.5 → 0(闭)
5. Rate control: sleep 补齐到目标频率
6. 循环直到 max_steps 或 Ctrl+C
```
