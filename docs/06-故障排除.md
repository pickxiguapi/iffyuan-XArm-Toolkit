# 故障排除

> 这篇解决的问题：机械臂/相机/SpaceMouse 出问题了怎么办。

## 急救操作

### 紧急情况处理优先级

1. **硬件急停**：按机械臂底座上的红色急停按钮（最快，物理断电）
2. **软件急停**：Web Demo 中按 Esc 键
3. **终端中断**：Ctrl+C 终止脚本（atexit 会自动调用 cleanup）**** 这是最常用的 ****

### 急停后恢复

```python
###
# 急停按钮按下后，需要手动旋转解除急停按钮
###

# 然后重新连接：
env = XArmEnv(addr="192.168.31.232")
obs = env.reset()  # 回到 Home
```

## 机械臂问题

### 连接失败

**症状**：`XArmAPI` 初始化卡住或报错

**排查**：
```bash
# 1. 检查网络连通性
ping 192.168.31.232

# 2. 检查端口（xArm 使用 502 端口）
nc -zv 192.168.31.232 502

# 3. 检查是否有其他程序占用连接
# xArm 同时只允许一个 SDK 连接！
# 关闭 xArm Studio、其他 Python 进程
```

**常见原因**：
- 网线没插好 / IP 不在同一网段
- 另一个程序已经连着机械臂（xArm Studio、另一个 Python 脚本）
- 机械臂还在自检（上电后等 5 秒）

### SDK 报错 error code

`XArmCommandError` 或日志里出现 `Error code X`：

| 错误码 | 含义 | 处理 |
|--------|------|------|
| 1 | 关节超限/碰撞检测 | reset() 回 Home，检查是否碰到东西 |
| 2 | 关节超速 | 降低 speed/joint_speed |
| 3 | 关节力矩超限 | 检查是否卡住，降低力矩 |
| 9 | 紧急停止 | 检查急停按钮是否按下 |
| 10 | 需要手动清除错误 | 调用 `arm.clean_error()` + `arm.motion_enable(True)` |
| 19 | 关节角度报错 | 机械臂可能需要重新标定 |
| 21 | 运动学错误 | 目标位姿超出工作空间 |

> **通用修复套路**：大多数错误可以通过以下序列恢复：
> ```python
> env.arm.clean_error()
> env.arm.clean_warn()
> env.arm.motion_enable(True)
> env.arm.set_mode(7)
> env.arm.set_state(0)
> ```
> 或者直接 `env.reset()` —— reset 内部会做上述操作。

### 机械臂不动了但没报错

**可能原因**：
1. Mode 不对 —— `env.reset()` 会自动修复
2. State 不是 0 —— `arm.set_state(0)`
3. 目标位姿和当前位姿一样（delta 模式下 action 全是 0）

### 运动方向和预期不一致

机械臂坐标系：
- **X**：机械臂正前方（远离底座方向）
- **Y**：机械臂左方
- **Z**：向上

SpaceMouse 映射考虑了夹爪旋转 90 度（yaw = -π/2）的坐标系变换。如果改了 Home 位姿，需要同步调整 `xarm_toolkit/teleop/spacemouse.py` 中 `act()` 方法里的旋转重映射逻辑：

```python
# spacemouse.py 中的映射（因为夹爪 yaw=-π/2，roll/pitch 交叉）
action[3] = -original_rot[1]   # roll  ← -pitch
action[4] = original_rot[0]    # pitch ←  roll
action[5] = original_rot[2]    # yaw   ←  yaw
```

## 力传感器问题

### 力传感器初始化失败

**症状**：`ft_sensor_enable() returned code X`

**处理**：
1. 确认力传感器硬件存在（内置 FT 传感器，不是所有型号都有）
2. 重新上电机械臂
3. 用 `use_force=False` 暂时绕过

### 力数据漂移

**症状**：`ext_force` 数值缓慢变大，即使没有外力

**处理**：
```python
# 每次采集前调零
env.reset_force_sensor_zero()
```

这是正常现象，力传感器有温漂。采集时每个 episode 开始前自动调零。

### force 数据全是 None

**原因**：`use_force=False`

**修复**：初始化时 `XArmEnv(use_force=True)`

## 相机问题

### 相机未检测到

```python
import open3d as o3d
print(o3d.t.io.RealSenseSensor.list_devices())
# 如果输出为空列表 → 相机未被系统识别
```

**排查**：
1. 确认 USB 3.0 接口（蓝色口）
2. 拔插 USB 线
3. `lsusb | grep -i intel`（Linux）或系统偏好 → USB 查看
4. 确认没有其他程序占用

### Serial 号不对

```python
# 列出所有相机及其 serial
import open3d as o3d
devices = o3d.t.io.RealSenseSensor.list_devices()
for d in devices:
    print(d)
```

### 两个相机只能打开一个

USB 带宽不足。把两个相机插在**不同的 USB 控制器**上（通常主板左右两侧的 USB 口属于不同控制器）。

### 画面全黑或颜色异常

- D435i / L515 启动后前几帧可能是黑的，属于正常预热
- 颜色顺序注意：Open3D 输出是 **RGB**，OpenCV 的 `imshow` 需要 **BGR**
  ```python
  cv2.imshow("x", cv2.cvtColor(rgb, cv2.COLOR_RGB2BGR))
  ```

## SpaceMouse 问题

### Linux 下无响应

```bash
# 1. 检查 spacenavd 驱动
sudo systemctl status spacenavd

# 2. 如果没运行
sudo systemctl start spacenavd

# 3. 检查设备
ls /dev/input/event*

# 4. 权限问题
sudo usermod -aG input $USER
# 重新登录后生效
```

## Web Demo 问题

### 端口被占用

```bash
# 查看谁占了 8080
lsof -i :8080

# 用其他端口
python scripts/web_control_demo.py --port 9090
```

### 页面打不开但脚本在跑

- 确认访问的是 `http://localhost:8080`（不是 https）
- 如果是远程机器，用 `http://<机器IP>:8080`

## 通用排查思路

遇到问题时按这个顺序排查：

1. **看终端日志**：`xarm_toolkit` 的日志会打印关键信息和错误码
2. **重启脚本**：Ctrl+C 终止，cleanup 会自动调用
3. **reset**：`env.reset()` 能解决大部分 SDK 状态问题
4. **重启机械臂**：关电源等 5 秒再开
5. **检查物理连接**：网线、USB、电源
6. **查 xArm 文档**：[xArm Python SDK 文档](https://github.com/xArm-Developer/xArm-Python-SDK)
